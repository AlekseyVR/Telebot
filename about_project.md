# 🛠️ Server Monitor Bot (Telegram) — Полная документация

## 📌 1. Общее описание проекта
**Server Monitor Bot** — это автономный инструмент на Python для непрерывного мониторинга и удаленного администрирования локального Windows-сервера. Бот объединяет в себе функции системного аналитика, сетевого диагноста и «цифрового сторожа» (Watchdog). 

Бот спроектирован для работы как в личных сообщениях с администратором, так и в **специализированных Telegram-группах**, обеспечивая прозрачность состояния сервера и оперативное оповещение обо всех сбоях для всей команды разработчиков.

---

## ⚙️ 2. Стек технологий и зависимости
Для обеспечения стабильной, отказоустойчивой и легковесной работы используются следующие компоненты:
* **Python 3.10+**: Основной язык разработки.
* **pyTelegramBotAPI (telebot)**: Взаимодействие с Telegram Bot API через синхронный Long Polling.
* **psutil**: Кроссплатформенная библиотека для глубокого анализа системных ресурсов (процессы, память, ядра, дисковое пространство).
* **subprocess, os, urllib**: Встроенные модули для выполнения команд ОС (ICMP-пинг) и проверки HTTP-доступов (выявление блокировок API).
* **python-dotenv**: Безопасная подгрузка токена бота из скрытого файла окружения.
* **json (built-in)**: Формат для хранения настроек, путей к логам, сетевых адресов и прав доступа.

## 🚀 3. Функциональные модули

### 📊 3.1. Ресурсы и «Железо» (System Health)
Бот анализирует состояние сервера и предоставляет данные в удобном виде:
* **CPU (Процессор)**: Общий процент загрузки всех ядер.
* **RAM (Оперативная память)**: Объем использованной и свободной памяти.
* **Disk Monitor (Мониторинг дисков)**: Контроль свободного места на системном (`C:\`) и пользовательском (`D:\`) дисках. Предотвращает остановку системы из-за переполнения.
* **Uptime**: Время работы сервера с момента последней загрузки Windows.

### 🌐 3.2. Сеть и API (Connectivity & Web Services)
Многоуровневая проверка связи сервера с внешним миром:
* **ICMP Ping (Маршрутизация)**: Проверка физической доступности узлов (например, `8.8.8.8`). Показывает, есть ли вообще интернет на сервере.
* **HTTP Checker (Анти-Блокировка)**: Умная проверка API-эндпоинтов (Telegram API, Gemini API). Бот имитирует браузерный запрос и анализирует HTTP-коды:
  * `200 OK` — Доступ открыт, API работает.
  * `403 Forbidden` — Провайдер, РКН или сам сервис заблокировал ваш IP (например, ограничения Gemini по региону).
  * `429 Too Many Requests` — Превышен лимит запросов к API.

### 🐕 3.3. Универсальный Watchdog (Сторож процессов)
Система автоматического слежения за критически важной инфраструктурой:
* **Скрипты** (`.py`, `.js`, `.bat`): Бот проверяет не только имя процесса, но и **аргументы командной строки**, чтобы точно идентифицировать работающий скрипт (например, `smart_bot.py`).
* **Приложения** (`.exe`): Прямой мониторинг баз данных и веб-серверов (`nginx.exe`, `redis-server.exe`).
* При падении процесса генерируется экстренный алерт.

### 📁 3.4. Дистанционная работа с логами (Log Management)
* **Команда `/getlogs`**: Бот сканирует конфигурационный файл, собирает указанные файлы логов и отправляет их в чат в виде документов (`.log` или `.txt`). Позволяет анализировать ошибки без RDP-доступа к серверу.
