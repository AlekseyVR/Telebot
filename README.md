# 🛡️ Telegram Server Monitor Bot

![Python Version](https://img.shields.io/badge/python-3.10%2B-blue.svg)
![Telebot](https://img.shields.io/badge/pyTelegramBotAPI-Latest-green.svg)
![Pydantic](https://img.shields.io/badge/pydantic-Strict_Validation-orange.svg)
![Platform](https://img.shields.io/badge/platform-Windows%20%7C%20Linux-lightgrey)
![License](https://img.shields.io/badge/license-MIT-green)

Легковесный, автономный и безопасный Telegram-бот для непрерывного мониторинга и удаленного администрирования серверов. Выполняет функции системного аналитика, сетевого диагноста и цифрового «сторожа» (Watchdog).

Бот спроектирован по паттерну Singleton с жесткой валидацией конфигов (Pydantic) и отлично подходит для работы как в личных сообщениях с администратором, так и в специализированных командных чатах.

---

## ✨ Ключевые возможности

* 📊 **Анализ ресурсов (System Health):** Отслеживание загрузки CPU, состояния RAM и свободного места на жестких дисках (`C:\`, `D:\`).
* 🌐 **Сетевая диагностика:** Умный ICMP-пинг и проверка HTTP/API эндпоинтов (с выявлением блокировок 403 и лимитов 429).
* 🐕 **Watchdog процессов:** Непрерывный мониторинг критической инфраструктуры (поддержка `.exe`, `.py`, `.bat`). Различает скрипты по аргументам запуска и **отслеживает точный Uptime каждого отдельного процесса**.
* 🔔 **Умные уведомления:** Фоновый цикл с тихими плановыми отчетами и громкими экстренными алертами при падении процессов.
* 📝 **Профессиональное логирование:** Встроенный логгер записывает все системные события и ошибки в файл `bot.log` с точными таймстемпами.
* ⏱ **Авторестарты (Self-healing):** Настраиваемая плановая перезагрузка самого скрипта для предотвращения утечек памяти.
* 📁 **Работа с логами:** Дистанционное получение любых файлов логов (включая собственный `bot.log`) напрямую в Telegram без необходимости RDP/SSH доступа.
* 🔄 **Удаленное управление:** Возможность экстренной перезагрузки сервера (Windows).

## 🚀 Быстрый старт

### 1. Клонирование и установка зависимостей
Убедитесь, что у вас установлен Python 3.10 или выше.

```bash
git clone [https://github.com/AlekseyVR/Telebot.git](https://github.com/AlekseyVR/Telebot.git)
cd Telebot
python -m venv venv

# Активация окружения (Windows)
venv\Scripts\activate
# Для Linux/macOS: source venv/bin/activate

pip install -r requirements.txt
```

### 2. Настройка безопасности (.env)
Создайте файл `.env` в корне проекта и добавьте ваши секретные ключи. **Этот файл не должен попадать в публичный репозиторий!**

```env
BOT_TOKEN=ваш_токен_от_BotFather
GEMINI_API_KEY=ваш_ключ_от_gemini_api
```

### 3. Конфигурация логики
Бот использует библиотеку `Pydantic` для строгой валидации типов. В корне проекта находится файл-шаблон `config.example.json`. Создайте его копию, переименуйте в `config.json` и укажите свои данные:

**Для Windows (в терминале):**
```cmd
copy config.example.json config.json
```

## 🕹️ Доступные команды

Все команды защищены строгой проверкой прав. Бот реагирует только на пользователей из списка `admins`.

| Команда | Описание |
| :--- | :--- |
| `/start` | Проверка авторизации и приветствие. |
| `/help` | Вывод подробной справки по всем функциям бота. |
| `/status` | Принудительная генерация полного среза состояния сервера (Ресурсы, Сеть, Watchdog с таймерами аптайма). |
| `/getlogs` | Отправка файлов логов из конфигурации в виде документов. |
| `/reboot` | Удаленная перезагрузка ПК (выполняется с задержкой 5 сек и оповещением в чат). |

## ⚙️ Бессмертный запуск (Windows 24/7)

Для обеспечения работы бота 24/7 в фоне, даже без входа пользователя в систему, рекомендуется использовать встроенный **Планировщик задач (Task Scheduler)** и скрипт-оболочку `run_monitor.bat`.

1. Отредактируйте `run_monitor.bat`, указав абсолютный путь к вашему виртуальному окружению (например, `".venv\Scripts\python.exe" main.py`).
2. Откройте **Планировщик задач** и создайте новую задачу.
3. **Общие:** Укажите пользователя `СИСТЕМА` (`SYSTEM`) и поставьте галочку "Выполнять с наивысшими правами" + "Скрытая задача".
4. **Триггеры:** Назначьте запуск "При запуске" (At startup). Рекомендуется добавить задержку в 1 минуту для инициализации сети.
5. **Действия:** Выберите запуск файла `run_monitor.bat`. **Критически важно:** в поле "Рабочая папка" укажите путь к корню проекта (без кавычек).
6. **Параметры:** Снимите галочку с "Останавливать задачу, выполняемую дольше 3 дней".

Скрипт `run_monitor.bat` в связке с внутренней функцией авторестарта (`planned_restart_sec` в конфиге) обеспечит абсолютную отказоустойчивость системы.
